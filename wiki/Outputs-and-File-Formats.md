# Outputs and File Formats

This document describes all file outputs generated by VocalTrack, including naming conventions, format specifications, and data filtering behavior.

---

## Output Directories

### Default Locations

- **Session recordings**: `recordings/` (relative to project root)
- **Benchmark reports**: `benchmarking/` (relative to project root)

### Customizing Output Directory

Output directory can be customized by editing `EXPORT_CONFIG` in `config.py`:

```python
EXPORT_CONFIG = {
    'save_recordings': False,  # Must be set to True to save files automatically
    'save_wav': True,
    'save_original_audio': True,
    'save_downsampled_audio': False,
    'save_csv': True,
    'output_dir': 'my_recordings',  # Custom directory name
}
```

## Session Naming Convention

All session files use timestamped naming:

```
speaker_YYYY-MM-DD_HHMMSS_[mode]_[type].[ext]
```

**Components:**
- `speaker`: Default speaker ID (hardcoded, can be modified in source)
- `YYYY-MM-DD`: Date (year-month-day)
- `HHMMSS`: Time (hour-minute-second, 24-hour format)
- `[mode]`: Mode identifier (`vowel` or `pitch`)
- `[type]`: Output subtype (`original` or `downsampled` for audio; `formants` or `pitch` for CSVs)
- `[.ext]`: File extension (`.wav`, `.csv`)

**Examples:**
- `speaker_2026-03-04_102534_vowel_original.wav`
- `speaker_2026-03-04_102534_vowel_downsampled.wav`
- `speaker_2026-03-04_102534_formants.csv`
- `speaker_2026-03-04_102539_pitch_original.wav`
- `speaker_2026-03-04_102539_pitch.csv`

**Note:** Current implementation uses fixed speaker ID "speaker". To customize, modify `exporter.create_session_name()` function.

## LiveVowel Outputs

### WAV File

**Filename**: `recordings/speaker_YYYY-MM-DD_HHMMSS_vowel_original.wav` (and/or `recordings/speaker_YYYY-MM-DD_HHMMSS_vowel_downsampled.wav`)

**Format:**
- **Encoding**: PCM (Pulse Code Modulation)
- **Bit depth**: 16-bit signed integer
- **Channels**: Mono (1 channel)
- **Sample rate**: Matches analysis config (typically 10000 Hz for max_formant=5000 Hz)
- **Endianness**: Little-endian (standard for WAV)

**Processing:**
- Audio is normalized during export (scaled to use ~97% of full int16 range)
- Normalization prevents clipping while maximizing signal-to-noise ratio
- Formula: `samples = samples × (32767 × 0.97 / max_abs_value)`

**Size estimation:**
- Sample rate: 10000 Hz
- Bit depth: 16 bits = 2 bytes
- Bytes per second: 10000 × 2 = 20000 bytes/sec = ~20 KB/sec
- 1 minute recording: ~1.2 MB
- 10 minute recording: ~12 MB

### CSV File

**Filename**: `recordings/speaker_YYYY-MM-DD_HHMMSS_formants.csv`

**Format**: UTF-8 encoded CSV with header row

**Columns:**

| Column | Type | Units | Description |
|--------|------|-------|-------------|
| `time_ms` | float | milliseconds | Timestamp relative to recording start |
| `f0` | float | Hz | Fundamental frequency (pitch) |
| `f1` | float | Hz | First formant frequency |
| `f2` | float | Hz | Second formant frequency |
| `f3` | float | Hz | Third formant frequency |
| `voicing` | boolean | True/False | Voicing status (True = voiced, False = unvoiced) |
| `track_number` | int | — | Trajectory segment identifier (unique per contiguous track) |

**Example rows:**

```csv
time_ms,f0,f1,f2,f3,voicing,track_number
20.0,123.4,567.8,1234.5,2345.6,True,1
40.0,125.1,570.2,1240.3,2350.1,True,1
60.0,127.8,575.6,1245.0,2355.7,True,1
```

**Data Filtering:**

Only frames meeting **all** of the following criteria are written to CSV:

1. `voicing == True` (voiced frames only)
2. `min_f0 <= f0 <= max_f0` (f0 within configured analysis range)
3. Formant values are not `None` or `NaN`

**This means:**
- Unvoiced segments (voiceless consonants, silence) are excluded
- f0 values outside the expected range are excluded
- Rows with failed formant extraction are excluded

**Typical filtering:**
- Original frames: 60/sec (assuming 20 ms chunks)
- After filtering: 30-50/sec (depending on speech content)
- Silence/unvoiced: 0/sec

**Track Number:**

`track_number` increments when:
- Voicing drops and resumes (new voiced segment)
- Formant trajectory becomes unstable
- Recording is stopped and restarted (Ctrl+R or Ctrl+H toggle)

Example: Three separate vowels → track_number values 1, 2, 3

**Time Resolution:**

- Determined by `chunk_ms` parameter (typically 20 ms)
- `time_ms` increments by `chunk_ms` for each frame
- Example: 20 ms chunks → rows at 0, 20, 40, 60, 80, ... ms

## LivePitch Outputs

### WAV File

**Filename**: `recordings/speaker_YYYY-MM-DD_HHMMSS_pitch_original.wav` (and/or `recordings/speaker_YYYY-MM-DD_HHMMSS_pitch_downsampled.wav`)

**Format:**
- **Encoding**: PCM
- **Bit depth**: 16-bit signed integer
- **Channels**: Mono
- **Sample rate**: Matches analysis config (typically 10000 Hz)
- **Endianness**: Little-endian

**Processing:**
- Audio is normalized using same algorithm as LiveVowel (97% of full scale)
- Each Space press/release creates a separate timestamped file

**Size estimation:**
- Same as LiveVowel (~20 KB/sec at 10000 Hz sample rate)

### CSV File

**Filename**: `recordings/speaker_YYYY-MM-DD_HHMMSS_pitch.csv`

**Format**: UTF-8 encoded CSV with header row

**Columns:**

| Column | Type | Units | Description |
|--------|------|-------|-------------|
| `track` | int | — | Track number (optional, only if multiple segments) |
| `time_ms` | float | milliseconds | Timestamp relative to recording start |
| `f0` | float | Hz | Fundamental frequency (pitch) |
| `voicing` | boolean | True/False | Voicing status |

**Example rows:**

```csv
track,time_ms,f0,voicing
1,20.0,123.4,True
1,40.0,125.1,True
1,60.0,127.8,True
```

**Note:** If only one recording segment exists (single Space press/release), `track` column may be omitted.

**Data Filtering:**

Only frames meeting **all** of the following criteria are written to CSV:

1. `voicing == True` (voiced frames only)
2. `min_f0 <= f0 <= max_f0` (f0 within configured analysis range)
3. f0 value is not `None` or `NaN`

**This means:**
- Unvoiced segments are excluded
- Out-of-range f0 estimates are excluded
- Failed pitch extractions are excluded

**Typical behavior:**
- User holds Space for 3 seconds
- Audio captures 3000 ms / 20 ms = 150 frames
- After filtering: ~100-120 frames (depending on voicing)
- Unvoiced periods (silence, consonants) excluded

**Push-to-Talk Workflow:**

Each Space down/up cycle:
1. Audio recording starts when Space is pressed
2. Pitch analysis runs in real-time
3. Audio recording stops when Space is released
4. WAV + CSV are immediately exported with current timestamp
5. New timestamp is generated for next Space press

**Result:** Multiple recording sessions create multiple file pairs with different timestamps.

## LiveSpectrogram Outputs

### Default Behavior

**No automatic file output** by default. LiveSpectrogram is designed as a visualization-only mode.

### Capturing Spectrogram Data

**Option 1: Screen recording**
- Use OS screen recording tools to capture visual output
- Windows: Xbox Game Bar (Win+G)
- macOS: QuickTime Player (File → New Screen Recording)
- Linux: RecordMyDesktop, SimpleScreenRecorder

**Option 2: Modify source to enable export**
- LiveSpectrogram has internal audio buffer (`self.audio_buffer`)
- Modify `LiveSpectrogram.py` to call `exporter.save_wav()` on quit
- Example modification:

```python
# In LiveSpectrogram.run() method, on quit:
if self.audio_buffer:
    filename = os.path.join('recordings', f'{self.session_name}.wav')
    exporter.save_wav(filename, self.audio_buffer, self.sample_rate)
```

**Option 3: External audio recording**
- Use system audio recording tool (Audacity, Ocenaudio)
- Record system audio while LiveSpectrogram runs

## Benchmark Output Files

VocalTrack includes built-in benchmarking tools (accessed via "Benchmarking" button in launcher) that generate comparison reports.

### Accuracy Benchmark

**Purpose**: Compare VocalTrack formant/pitch extraction against Parselmouth (Praat-based reference).

**Output files:**

1. **Formant comparison** (`benchmarking/formant_comparison.csv`)

Columns:
- `time_ms`: Timestamp
- `native_f1`, `native_f2`, `native_f3`: VocalTrack formant estimates
- `parselmouth_f1`, `parselmouth_f2`, `parselmouth_f3`: Parselmouth formant estimates
- `error_f1`, `error_f2`, `error_f3`: Absolute differences (Hz)

2. **Pitch comparison** (`benchmarking/pitch_comparison.csv`)

Columns:
- `time_ms`: Timestamp
- `native_f0`: VocalTrack pitch estimate
- `parselmouth_f0`: Parselmouth pitch estimate
- `error_f0`: Absolute difference (Hz)

3. **Accuracy report** (`benchmarking/accuracy_report.md`)

Markdown report containing:
- Mean absolute error (MAE) for each formant and f0
- Root mean square error (RMSE)
- Correlation coefficients
- Summary statistics

### Timing Benchmark

**Purpose**: Measure computational performance of VocalTrack analysis methods.

**Output files:**

1. **Timing comparison** (`benchmarking/timing_comparison.csv`)

Columns:
- `method`: Analysis method name (`native`, `parselmouth`, etc.)
- `formant_time_ms`: Formant analysis duration (milliseconds)
- `pitch_time_ms`: Pitch analysis duration (milliseconds)
- `total_time_ms`: Total analysis duration (milliseconds)
- `frames_processed`: Number of frames analyzed
- `fps`: Effective frame rate (frames per second)

2. **Timing report** (`benchmarking/timing_report.md`)

Markdown report containing:
- Summary statistics for each method
- Speed comparison (which method is faster)
- Real-time factor (RTF) analysis

3. **Method-specific timing** (e.g., `benchmarking/timing_results_native.csv`)

Per-frame timing data for detailed analysis.

Columns:
- `frame_number`: Frame index
- `formant_time_ms`: Time to analyze formants
- `pitch_time_ms`: Time to analyze pitch
- `total_time_ms`: Total time
- `cumulative_time_ms`: Running total

## File Size Management

### Estimating Storage Requirements

**Per recording session (LiveVowel/LivePitch):**

| Duration | WAV size (10 kHz) | CSV size (est.) | Total |
|----------|-------------------|-----------------|-------|
| 10 seconds | ~200 KB | ~10 KB | ~210 KB |
| 1 minute | ~1.2 MB | ~60 KB | ~1.3 MB |
| 5 minutes | ~6 MB | ~300 KB | ~6.3 MB |
| 10 minutes | ~12 MB | ~600 KB | ~12.6 MB |

**Note:** CSV size varies significantly based on:
- Amount of voiced speech (unvoiced periods excluded)
- f0 range filtering (out-of-range frames excluded)
- Speech rate and pausing

### Archiving and Compression

**Compressing WAVs:**

WAV files compress very well with lossless compression:

```bash
# Zip compression
zip recordings.zip recordings/*.wav
# Typical compression ratio: 50-60%

# 7-Zip compression (better)
7z a recordings.7z recordings/*.wav
# Typical compression ratio: 60-70%
```

**Compressing CSVs:**

CSVs are text files and compress extremely well:

```bash
# Gzip compression
gzip recordings/*.csv
# Typical compression ratio: 80-90%
```

**Batch compression (preserving structure):**

```bash
# Archive entire recordings folder
tar -czf recordings_archive.tar.gz recordings/
```

## Troubleshooting Output Issues

### No Files Generated

**Problem**: Launched mode, spoke, quit, but no files in `recordings/` folder.

**Possible causes:**

1. **Mode doesn't export by default**: LiveSpectrogram and LiveSpectrum are visualization-only
   - Solution: Use LiveVowel or LivePitch for automatic export

2. **No voiced frames detected**: All frames filtered out due to voicing/f0 range
   - Solution: Check that f0 is within configured range, adjust `min_f0`/`max_f0`

3. **Recording not started**: For LiveVowel, must press Ctrl+R (or Ctrl+H to hide help) to enter recording state
   - Solution: Verify you pressed Ctrl+R or Ctrl+H before speaking

4. **Export disabled in config**: `save_recordings`, `save_wav`, or `save_csv` set to False (or not enabled in Recording Settings)
   - Solution: Open Recording Settings and check "Save recordings", or check `EXPORT_CONFIG` in `config.py`

5. **Write permissions issue**: Cannot write to `recordings/` folder
   - Solution: Check folder permissions, run as administrator if needed

### Empty CSV Files

**Problem**: CSV file exists but contains only header row (no data).

**Possible causes:**

1. **No voiced speech**: Only unvoiced sounds produced (whisper, fricatives)
   - Solution: Produce voiced sounds (vowels, nasals, /v/, /z/) 

2. **f0 out of range**: All f0 estimates outside `min_f0` to `max_f0` range
   - Solution: Widen f0 range in Analysis Settings

3. **RMS threshold too high**: `min_rms_db` filters out all frames
   - Solution: Lower threshold (more negative, e.g., -70 dB)

4. **Formant extraction failure**: All frames return None for formants
   - Solution: Try switching `formant_method` from native to parselmouth

### WAV Files Too Quiet or Too Loud

**Problem**: Audio level is too low or clipping.

**Cause**: VocalTrack normalizes to 97% of full scale, but original recording may have issues.

**Solutions:**

1. **Too quiet**: Increase microphone gain in OS settings before recording
2. **Too loud (clipping)**: Decrease microphone gain in OS settings
3. **Post-processing**: Use Audacity or similar to adjust levels after export

### CSV Timestamp Gaps

**Problem**: CSV shows timestamp jumps (e.g., 0, 20, 40, 100, 120...).

**Cause**: Normal behavior—unvoiced frames are excluded from CSV.

**Explanation**: Gaps correspond to:
- Unvoiced consonants (/p/, /t/, /k/, /s/, /f/, etc.)
- Silence between words
- f0 estimates outside configured range

**This is intentional**: CSV contains only voiced, valid formant/pitch data.

## Best Practices

1. **Archive regularly**: Move recordings to dated subfolders (e.g., `recordings/2026-03-04/`)
2. **Compress old data**: Use gzip or zip to save space on archived sessions
3. **Document sessions**: Keep a log file with speaker info, recording conditions, etc.
4. **Verify immediately**: Check that files exist and contain data right after recording
5. **Use consistent settings**: Keep analysis settings constant for within-group comparisons
6. **Export settings with data**: Copy `.VocalTrack_settings.json` to archive with recordings
7. **Test pipeline early**: Verify you can import and analyze CSV/WAV before large studies
